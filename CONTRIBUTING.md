# Contributing to github-final-project

First off, thank you for considering contributing to this Simple Interest Calculator! It’s people like you who make the open-source community such a great place to learn, inspire, and create.

## How Can I Contribute?

### 1. Reporting Bugs
If you find a calculation error or a crash:
* **Check the Issue Tracker** to see if the bug has already been reported.
* If not, **open a new issue**. Include a clear title, a description of the steps to reproduce the bug, and the expected vs. actual output.

### 2. Suggesting Enhancements
Want to add a feature (like calculating Compound Interest)? 
* Open an issue with the tag `enhancement`.
* Explain why this feature would be useful to other users.

### 3. Your First Code Contribution
1. **Fork the Repository**: Click the 'Fork' button at the top right of this page.
2. **Clone your Fork**: 
   `git clone https://github.com/YOUR-USERNAME/github-final-project.git`
3. **Create a Branch**: 
   `git checkout -b feature/YourFeatureName`
4. **Make Changes**: Ensure your code follows the project style.
5. **Commit**: Use clear messages like `git commit -m "Add error handling for zero interest rate"`.
6. **Push**: `git push origin feature/YourFeatureName`
7. **Open a Pull Request (PR)**: Go to the original repository and click "New Pull Request."

## Style Guidelines
* **Code**: Please ensure your variables are descriptive (e.g., use `time_period` instead of `t2`).
* **Documentation**: Update the README if you change how the inputs work.
